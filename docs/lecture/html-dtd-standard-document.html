# 표준 호환모드

문서 유형 정의(Document Type Definition)를 말하며 브라우저의 렌더링 모드를 표준으로 작동하게 만들어 줍니다. DTD는 HTML 문서의 반드시 최상 위에 위치해야 합니다.

코드 간 들여쓰기를 하는 이유

HTML 요소 간 관계(부모, 자식, 형제)를 구분하기 용이 함에 들여쓰나, 브라우저는 코드가 한 줄 이어도 해석하는데 아무런 문제가 없습니다. 참고로 코드 에디터의 설정에 따라 tab은 간격 설정이 달라집니다. 영상 강의에서는 space 2칸이 사용됩니다.

# Q & A

수강생과 주고 받은 질문/답변 입니다. 참고하세요.

Q1

body가 html의 last child 라고 하셨는데요.. 저는 가끔 닫는 body 태그 밑에 외부 js들을 불러오는데요 외부 js를 그럼 body 최하단 에서 불러오는 것이 표준인건가요?

A1

웹 표준 기술 규격에 따르면 html 요소는 오직 1개의 head와 뒤 따르는 1개의 body 요소만 자식으로 포함할 수 있습니다. 

A <head> element followed by a <body> element.

OO님께서는 이러한 표준 규칙을 잘 몰랐기 때문에 지금까지 그러했을 수 있습니다. 이번 강의를 들으시면서 잘못된 부분을 바로 잡으실 수 있으면 좋겠습니다. ^ㅡ^ 간단하게 요약하면 <head> 내부 또는 <body> 내부에 작성해야 합니다.
