# HTML 문법 유효성 검사(Validator)

HTML 문서의 문법이 올바르게 작성 되었는지 확인하는 방법에 대해 공부해봅니다. 그리고 각 괄호(Angle Bracket, <>) 모양의 문자를 브라우저가 태그로 인식하지 않도록 이스케이프(Escape) 처리하는 엔티티(Entity) 코드에 대해서도 살펴봅시다.

# Q & A

수강생과 주고 받은 질문/답변 입니다. 참고하세요.

Q1

, 와 . 또는 ! ? " 등 특수문자는 엔티티코드로 쓰는게 좋은 건가요?
최대한 엔티티코드를 활용할려고 하는데.. 영상을 보니 < > 내용만 있어서 여쭤봅니다!!

A1

, 와 . 또는 ! ? " 등은 특수 문자가 아니라서 그대로 사용하셔도 됩니다.

HTML 태그로 브라우저가 해석하는데 문제가 될 수 있는 < >코드는 엔티티 코드로 사용해야 합니다.
그 외 ©, ·, €, ♥, ½ 와 같은 키보드 자판에 없는 심볼을 HTML 문서에 사용할 때 엔티티 코드를 사용할 수 있습니다.

Q2

카피라이트는 엔티티를 사용해서 하는건가요?
웹 표준검사기에서는 태그안에 들어가도 문법상 오류는 없다고하네요.

A2

엔티티 코드를 사용하는 이유는 크게 2가지입니다.

하나는 HTML 예약어(< >)와 충돌되는 문제를 해결하기 위함이고, 다른 하나는 키보드에 없는 문자를 출력하기 위함 입니다. © 기호는 키보드에 없는 문자 이지만, 엔티티 코드를 사용하면 화면에 출력할 수 있습니다. © 기호를 사용한다고 해서 문법 오류는 아닙니다. ^^
